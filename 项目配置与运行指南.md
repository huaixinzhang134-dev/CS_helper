# CS Match Pro 配置与运行指南

## 一、项目概览

```
cs/
├── miniprogram/          # 微信小程序前端
├── server/               # Express 后端（Node.js + MySQL）
├── crawler/              # HLTV / 5eplay 爬虫
├── scripts/              # 数据导入工具
└── .github/workflows/    # GitHub Actions 自动化
```

## 二、环境变量配置

### 2.1 后端（`server/.env`）

复制 `server/.env.example` 为 `server/.env`，填写以下变量：

```env
# ---- 数据库（Railway MySQL）----
DB_HOST=hayabusa.proxy.rlwy.net
DB_PORT=16612
DB_USER=root
DB_PASS=<你的Railway数据库密码>
DB_NAME=railway

# ---- 微信小程序登录 ----
WX_APPID=<你的小程序AppID>
WX_SECRET=<你的小程序AppSecret>

# ---- 用户认证密钥（自定义随机字符串）----
AUTH_SECRET=<随机字符串>

# ---- 爬虫同步 Token（自定义，与 GitHub Secrets 一致）----
SYNC_TOKEN=<随机字符串>

# ---- 服务端口 ----
PORT=3000
```

### 2.2 GitHub Actions Secrets

在 GitHub 仓库 `Settings → Secrets and variables → Actions` 中配置：

| Secret | 说明 |
|--------|------|
| `DB_HOST` | Railway MySQL 地址 |
| `DB_PORT` | Railway MySQL 端口 |
| `DB_USER` | Railway MySQL 用户 |
| `DB_PASS` | Railway MySQL 密码 |
| `DB_NAME` | Railway MySQL 库名 |
| `SYNC_TOKEN` | 爬虫同步 Token（与 `server/.env` 一致） |

### 2.3 微信小程序

`miniprogram/config.ts` 中配置后端 API 地址：

```ts
export const API_BASE = 'https://你的后端域名';
export const STATIC_BASE = 'https://你的静态资源域名';
```

## 三、本地运行

### 3.1 后端

```bash
cd server
npm install
node app.js
# 服务运行在 http://localhost:3000
```

### 3.2 爬虫

```bash
cd crawler
npm install

# 全量爬取选手信息（Active + Retired）
node player_data.js

# 分片爬取（用于 CI，7个分片并行）
node player_data.js --chunk 0  # 0~6

# 检查/修正选手位置 + 复出检测（需 DB 密码环境变量）
DB_PASS=xxx node player_check.js --quick --db

# 爬取世界排名
node crawl_ranking.js
```

### 3.3 数据导入

```bash
# CloudBase 导入（旧版，如仍使用）
cd scripts
node import-players.js

# 排名导入（如使用 Railway MySQL）
node import_ranking.js
```

## 四、GitHub Actions 工作流

| 工作流 | 频率 | 说明 |
|--------|------|------|
| `players.yml` | 每月 1 日 5:00 | 全量爬取选手信息 + 导入 Railway |
| `player-check.yml` | 每 3 天 5:00 | 检查选手位置 + 复出检测 |
| `ranking.yml` | 按需/定时 | 爬取 Valve 世界排名 |
| `crawler.yml` | 每 10 分钟 | 比赛数据同步（5eplay → Railway） |

## 五、数据库结构

### 核心表

| 表 | 说明 |
|----|------|
| `player` | 选手信息（含 status 字段） |
| `team` | 战队信息 |
| `team_member` | 战队-选手关联 |
| `team_ranking` | Valve 世界排名 |
| `matches` | 比赛数据 |
| `match_players` | 比赛选手个人数据 |
| `users` | 用户（微信登录） |
| `player_comments` | 选手评论 |

### player.status 枚举

| 值 | 含义 |
|----|------|
| `active` | 现役选手 |
| `coach` | 教练 |
| `free_agent` | 自由人（无队） |
| `retired` | 已退役 |
| `unknown` | 待标注 |

## 六、关键 API

| 路由 | 说明 |
|------|------|
| `GET /api/players` | 选手列表（分页） |
| `GET /api/players/:id` | 选手详情 |
| `GET /api/players/pool?difficulty=hell` | 猜一猜选手池 |
| `GET /api/players/random-by-difficulty?difficulty=hard` | 随机目标选手 |
| `GET /api/players/search?q=NiKo` | 搜索选手 |
| `POST /api/matches/sync` | 比赛数据同步（需 Bearer Token） |
| `POST /api/users/login` | 微信登录 |
| `POST /api/guess/record` | 提交竞猜记录 |

## 七、常见问题

**Q: 本地爬虫报数据库连接失败？**
设置 `DB_PASS` 环境变量后再运行，或通过 `server/.env` 自动加载。

**Q: GitHub Actions 爬虫失败？**
检查 Actions Secrets 中 `DB_PASS` 是否与 Railway 一致。

**Q: 小程序无法连接后端？**
确认 `miniprogram/config.ts` 中 `API_BASE` 指向正确的后端地址，且微信公众平台已将该域名加入 request 合法域名。
